# FocusOS

Mike-oriented daily attention briefing.

The MVP answers one question in under 60 seconds:

> What deserves my attention today?

Phase 1 starts with finance, then adds user-defined topics that can be briefed from structured sources or AI web-search summaries. No trading, order entry, social features, notifications, mobile app, browser extension, or multi-user features.

## First Deliverable

One page.

- Today's Attention
- Financial Attention
- Topic Briefings

Manual finance CSV imports are enough for the first structured source. Topic briefings are generated by the morning job. With `OPENAI_API_KEY` configured, unstructured topics use the OpenAI Responses API with web search. Without a key, deterministic fallback briefings keep the app usable locally.

## Run Locally

```bash
docker compose up --build
```

Then open:

- Web: http://localhost:5173
- API: http://localhost:8000/api/briefing

The API seeds Mike-oriented sample holdings and default topics when the database is empty, so the homepage has a useful shape before the first import.

## CSV Import

Import endpoint:

```bash
curl -F "file=@holdings.csv" "http://localhost:8000/api/import/holdings?source=Fidelity&replace=true"
```

CSV uploads are limited to `FOCUSOS_MAX_IMPORT_BYTES`, defaulting to 1 MiB, and must be uploaded as a `.csv` file.

The importer accepts common column names such as:

- `symbol`, `ticker`
- `name`, `description`
- `quantity`, `qty`, `shares`
- `price`, `last price`, `current price`
- `market value`, `current value`, `value`
- `cost basis`, `total cost`
- `account`, `account name`
- `asset class`, `type`, `category`

## Morning Briefing Job

Manual trigger:

```bash
curl -X POST http://localhost:8000/api/jobs/morning-briefing
```

The response includes a `job_id`. Check status with:

```bash
curl http://localhost:8000/api/jobs/morning-briefing/<job_id>
```

The intended local schedule is:

- 06:00 jobs start
- 06:05 structured data collected
- 06:10 AI topic briefings generated
- 06:15 daily briefing assembled
- 07:00 ready to read

Docker Compose includes a `scheduler` service that triggers the morning job daily at `MORNING_JOB_TIME`, defaulting to `06:00` with `TZ=America/New_York`.

The scheduler service uses APScheduler.

Optional AI web search:

```bash
OPENAI_API_KEY=... docker compose up --build
```

The default model is `gpt-5.5`; override with `OPENAI_MODEL` if needed.

Each topic-generation call has a provider timeout controlled by `OPENAI_REQUEST_TIMEOUT`, defaulting to `20` seconds. If a call fails or times out, FocusOS stores the deterministic fallback briefing for that topic.

Local Codex CLI provider:

```bash
AI_PROVIDER=codex_cli CODEX_CLI_TIMEOUT=90 .venv/bin/uvicorn app.main:app --app-dir backend --host 127.0.0.1 --port 8000
```

This uses `codex exec --search` through the locally authenticated Codex CLI instead of direct OpenAI API calls. It is intended for local development, not the Docker container.

## Internal Status

Connector health is internal-only:

```bash
curl http://localhost:8000/api/internal/source-status
```

Do not surface source diagnostics on the homepage.

## Security Configuration

FocusOS is a local-first personal app. Public read endpoints stay unauthenticated for local development, but the API now applies browser hardening headers and rejects state-changing browser requests from origins outside `FOCUSOS_CORS_ORIGINS`.

Defaults:

- `FOCUSOS_CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173`
- `FOCUSOS_MAX_IMPORT_BYTES=1048576`
- `FOCUSOS_ENABLE_HSTS=false`

Set `FOCUSOS_INTERNAL_API_KEY` to require `X-FocusOS-Key` on internal operational routes:

- `POST /api/jobs/morning-briefing`
- `GET /api/internal/source-status`

The Compose scheduler passes `FOCUSOS_INTERNAL_API_KEY` through automatically when it is configured. Enable `FOCUSOS_ENABLE_HSTS=true` only behind HTTPS.

## Failure Handling

FocusOS keeps deliberate resilience for local development and source outages, but production-path failures should be visible:

- Holdings CSV imports fail with `400` when numeric portfolio fields contain malformed values. Blank values, `--`, and `n/a` still import as zero because broker exports commonly use those placeholders.
- Structured source outages or invalid provider payloads are recorded in `/api/internal/source-status`. The morning briefing continues with the last stored data or without that source's attention items.
- Database write failures are not treated as source outages. They roll back and propagate so the morning job is marked failed.
- If no AI provider is configured, unstructured topics use deterministic fallback briefings and `/api/internal/source-status` records `AI Topic Briefings` as `skipped`.
- If an explicitly configured AI provider fails, FocusOS stores fallback topic briefings, logs the provider exception with stack trace, and records `AI Topic Briefings` as `partial` or `error`.
- The scheduler keeps running after an API trigger failure and logs the exception in the scheduler service logs.

## Recommendation Details

Homepage recommendations are clickable. The detail payload is available at:

```bash
curl http://localhost:8000/api/recommendations/<detail_id>
```

Detail views explain why the recommendation was generated, raw/source data, AI processing when applicable, and suppressed signals.

## MVP Rule

Every attention item must answer:

1. Why should I care?
2. Why now?
3. Can I act?

If a finance signal or topic briefing cannot satisfy all three, it should not appear in Today's Attention.
