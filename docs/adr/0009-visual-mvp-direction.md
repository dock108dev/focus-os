# ADR 0009: Visual MVP direction

## Status

Superseded by ADR 0010.

## Context

FocusOS should not feel like a SaaS dashboard, finance tracker, AI chat, or admin console. The visual MVP prompt describes the target as:

> Presidential Daily Brief x newspaper x personal intelligence report.

The app should feel like opening a private morning briefing packet.

## Decision

Use a mobile-first briefing/feed structure instead of a multi-column dashboard grid.

Visual rules:

- warm paper backgrounds
- dark charcoal text
- restrained military green, navy, and burgundy accents
- serif typography for masthead and headlines
- sans-serif metadata and body support
- no sidebar
- no glassmorphism
- no neon, bright gradients, or generic AI dashboard styling
- recommendations appear as report entries
- details appear as evidence appendices

## Consequences

The homepage now reads as a single morning document:

- masthead
- primary attention entries
- recommended actions
- financial intelligence
- topic briefings
- inline evidence details

This supports the product goal: reduce the urge to check multiple apps by making the morning briefing feel complete and trustworthy.
